# 一个设计师用 AI 搭建个人网站的完整记录

## 背景

我叫 Bear Liu，做了十几年产品设计，之前在 Xero 做 Product Designer，后来开始做 Fractional Design Partner——帮早期创业团队做产品设计决策和执行，不用全职招人，按月合作。

我一直想给这个服务做一个专门的落地页。设计稿在 Figma 里早就画好了，但写代码这件事一直拖着。找外包要沟通成本，自己写又不够熟练。直到我开始用 Claude Code，发现这个事情可能没那么难。

## 2026 年 2 月 12 日，一天搞定

对，整个网站从零到上线，是在一天之内完成的。不是加班熬夜那种，是正常节奏推进，中间还处理了别的事情。下面是完整的过程。

---

### 上午 9:30 — 从一个空文件夹开始

我在终端里打开 Claude Code，把 Figma 设计稿的链接丢给它，说："帮我基于这个设计稿做一个网站。"

Claude 做的第一件事不是写代码，而是先制定了一个分阶段的实施计划。这个挺重要——它不是上来就乱写，而是先把整个项目拆成了 6 个阶段：连接 Figma、初始化项目、实现页面、响应式适配、Git 版本控制、Vercel 部署。

我看了一下计划，觉得靠谱，就说"开始吧"。

### 9:30–9:35 — 连接 Figma 设计稿

第一步是让 Claude 能直接读取我的 Figma 文件。通过 Figma 的 MCP（Model Context Protocol）服务，Claude 可以直接访问设计稿，提取颜色、字体、间距等设计参数，还能直接下载图片素材。

这一步很顺利，几分钟就搞定了。Claude 自动提取出了所有的设计 token：

- 品牌黄 `#F2CC0D`
- 棕色 `#A2845E`
- 各种灰度
- 字体大小和行高的完整体系

### 9:35–9:55 — 初始化项目 + 实现桌面端布局

Claude 用 Next.js + Tailwind CSS 创建了项目，然后开始逐个区块实现页面。整个网站有 9 个区块：

1. 导航栏
2. Hero 大图
3. 社会认证（合作品牌 Logo）
4. 痛点描述
5. 服务策略（路线图、Slack 协作、异步交付）
6. 案例研究（MovesMethod、Xero）
7. 定价方案
8. CTA 预约按钮
9. 页脚

不到 20 分钟，桌面端的完整布局就出来了。25 张图片和 SVG 素材全部从 Figma 提取并加载到位。我在浏览器里打开 localhost:3000，页面已经能看了。

### 下午 — 桌面端精调

吃完午饭回来，我对着 Figma 设计稿仔细看了一遍，发现一些间距和定位还需要微调。跟 Claude 说了之后，它很快就做了修正。还有一个小问题：Strategy 区块的图片阴影带了一点蓝色色调，不是设计稿里的纯灰色，也一并修了。

### 下午 — 响应式适配

这是工作量最大的一步。设计稿是按 960px 桌面宽度画的，但网站需要在手机（375px）和平板（768px）上也能正常显示。

Claude 把所有 9 个区块从绝对定位改成了 Flex 布局，加了响应式断点。几个关键的处理：

- **Hero 区块**：桌面端和手机端用了不同的背景图。手机端需要竖版裁切，Claude 帮我处理了这个。
- **导航栏**：提取成了独立的客户端组件，加了汉堡菜单，在小屏幕上自动切换。
- **Strategy 区块**：原来用了 13 个绝对定位，全部改成了三个 Flex 块，加上 `aspect-ratio` 来控制图片比例。
- **定价卡片**：从横排变成竖排堆叠，按钮尺寸也做了移动端适配（44px 触摸目标）。

过程中还顺手修了一些视觉问题：黄色高亮背景原来是用 SVG 做的，缩放时圆角会变形，Claude 直接改成了 CSS 背景色 + 圆角，干净多了。

### 傍晚 — 动画和微交互

页面结构搞定后，我想加一些让页面更有生气的细节。Claude 加了：

- 滚动触发的渐显动画（每个区块滚动到视口时淡入 + 上移）
- Hero 区块的加载动画（标题、描述、按钮依次出现）
- 卡片和图片的 hover 效果（缩放、阴影、位移）
- 导航栏 Logo 的 hover 缩放

这些动画都很克制，不会过度，但能让整个页面感觉更精致。

### 晚上 — 吸顶导航 + 毛玻璃效果

接下来我想让导航栏在滚动时固定在顶部，这样用户随时都能点到"Book Intro"按钮。Claude 加了吸顶效果和毛玻璃背景——页面顶部时导航是透明的，往下滚动后变成半透明白色加模糊效果，很像 Apple 和 Stripe 网站的风格。

**这里碰到了第一个 bug**：导航栏加了 `sticky` 之后不生效，滚动时不会固定在顶部。我截了个图给 Claude 看，它查了一下代码，发现是页面最外层的 `<div>` 上有一个 `overflow-x-hidden`，这个 CSS 属性会破坏 `position: sticky` 的行为。

解决方案很简单：把 `overflow-x-hidden` 从那个 div 移到 `<body>` 上。因为 body 本身就是滚动容器，放在它上面不会影响 sticky。改完之后导航栏就正常吸顶了。

这种问题如果自己查，可能要花不少时间搜 Stack Overflow。但 Claude 几秒钟就定位到了原因，这是 AI 辅助开发最有价值的地方之一。

### 晚上 — 接入真实链接

最后一步是把所有链接换成真实地址：

- 所有"Book Intro"按钮 → Calendly 预约链接
- "Check an example audit" → Notion 审计样例页面
- "UX New Zealand Speaker"徽章 → UXNZ 演讲者页面
- 页脚的 LinkedIn、X、邮箱 → 我的真实账号

还给合作品牌的 Logo 加了 hover 缩放效果。

### 晚上 10 点 — 推送上线

所有代码提交到 GitHub，Vercel 自动触发部署。几十秒后网站就上线了。

之后我在 Vercel 后台配置了自定义域名 `bearliu.com`，在域名注册商那边改了 DNS 记录，等了几分钟，SSL 证书自动生效。

打开 bearliu.com，一切正常。

## 碰到的问题和解决方式

整个过程不是完全顺风顺水，有几个值得记录的问题：

### 1. 目录名有空格

项目文件夹叫"FDP website"，中间有空格。`create-next-app` 不接受带空格的目录名，Claude 用了 `--name` 参数指定了一个不带空格的包名来绕过这个限制。之后所有终端命令都需要用引号包裹路径。

### 2. Tailwind CSS v4 的配置方式变了

Tailwind 从 v3 升级到 v4 后，主题配置不再用 `tailwind.config.js`，而是直接在 CSS 文件里用 `@theme inline` 语法。Claude 一开始就用了正确的方式，没有走弯路。

### 3. Sticky 导航不生效

上面提到的 `overflow-x-hidden` 问题。CSS 规范规定，sticky 元素和滚动容器之间的任何祖先元素如果设置了 `overflow: hidden/auto/scroll`，sticky 就会失效。这是一个很经典的"坑"。

### 4. SVG 背景缩放变形

设计稿里的黄色高亮背景用了 `preserveAspectRatio="none"` 的 SVG，在不同宽度下圆角会被拉伸变形。Claude 直接用 CSS 的 `background-color` + `border-radius` 替代了这些 SVG，效果更好也更好维护。

## 一些感想

这个项目让我对 AI 辅助开发有了更直观的体感。几个核心感受：

**设计师自己做开发变得可行了。** 以前"设计师会写代码"意味着要学很多工程知识。现在有了 Claude Code，我只需要知道我想要什么效果，能看懂代码大概在做什么就够了。真正的工程实现可以交给 AI。

**Figma 到代码的链路打通了。** Claude 能直接读取 Figma 设计稿，提取设计参数和素材，这省去了最繁琐的"翻译"工作。

**排错效率很高。** sticky 不生效这种问题，自己查可能要半小时到一小时。Claude 几秒就定位到了根因。它对 CSS 规范的理解比大多数人都深。

**迭代速度很快。** "加个 hover 效果"、"把这个链接换一下"、"导航栏做成吸顶的"——每个需求基本是说完就做完，不需要等排期或者来回沟通。

当然，这不意味着不需要人的判断。整个过程中我需要做的是：决定网站需要哪些内容、审核设计还原度、测试交互体验、提供真实的业务链接。AI 负责执行，我负责方向和品质。

这可能就是 Fractional Design Partner 这个服务本身想传达的理念——你不需要什么都自己做，找到对的合作方式，就能高效地把事情做好。
